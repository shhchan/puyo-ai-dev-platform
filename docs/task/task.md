# 依頼内容

ぷよぷよのゲームエンジンは大体形になってきたのですが，まだ細かいところが気になるのでいくつか修正をしてほしいです．

## 床キックおよびまわしの仕様修正

ぷよぷよの回転で発生する床キックが，0.5 マス単位の高さにあるときに 0.5 マス分しかキックが発生していない状態になっているので，ちゃんと 1.0 マス分上へ上がるようにしてください．
これによって「まわし」という操作ができるようになります．
例えば，左から2列目に10段，左から4列目に11段ぷよが積み重なっているとして，操作中のぷよが左から2列目の高さ 10.5 のところ（既に積まれている10段の高さより 0.5 マス高いところ）にあるとします．
この状態にある操作中の組ぷよで回転を行なって回転状態が「下」になると，組ぷよが1段上へ押し上げられて，高さ 11.5 になります．
高さ 11.5 になると4列目の11段の高さを超えられるようになるので，その直後に組ぷよを右へ移動させることで4列目より右側へ組ぷよを持っていけるようになります．

もし，左から2列目に10段，左から4列目に11段という状況は同じだとして，操作中の組ぷよが左から2列目の高さ 10.0 のところ（既に積まれている10段の高さにちょうど接するところ）にあるとします．
この状態にある操作中の組ぷよで回転を行なって回転状態が「下」になると，組ぷよが1段上へ押し上げられて，高さ 11.0 になります．
高さ 11.0 だと左から4列目の11段の高さはギリギリ超えられないというのがぷよぷよの仕様なので，この状態では組ぷよを右へ移動させようとしても4列目の壁を超えられない状態が正しいものとして扱われる必要があります．

## 隣接する同色ぷよの連結表現の実装

ぷよぷよでは隣接する同色のぷよは連結して表示されるようになります．
現状だとすべて丸で表現されているので，設置されたフィールド上のぷよでは連結したような見た目になるようにしてください．
ただし，13段目・14段目へ配置されたぷよへの連結表現や13段目・14段目の中での連結表現は行なわれないようにする必要があります．

## スコア表示の修正

現在のスコア表示はフィールド上部に表示するようになっています．
実際のぷよぷよではフィールドのすぐ下にスコア表示がされているので，そうなるようにしたいです．
スコアは8桁表示の左0埋めで，数字自体は 1P 側は右寄せ，2P 側は左寄せになります．
まだ 2P 側の実装はされていないので，今回は右寄せになります．

連鎖が発生していないときは現在のスコアを表示し続けるのですが，連鎖が発生しているときは N 連鎖目で可算されるスコアが一時的に表示されるようになる必要があります．
具体的な N 連鎖目のスコア算出については実装を参照してもらうとして，

```
N 連鎖目のスコア = A x B
A = 10 x (消したぷよの数)
B = (連鎖ボーナス) + (連結ボーナスの合計) + (色数ボーナス)
```

のような形になっています．
N 連鎖目のアニメーション中はこの `AxB` をスコアの欄に表示にしてほしいです．
`B` は3桁左半角スペース埋めで，その左に `x` があって，その左に `A` が3桁左半角スペース埋めになるような文字列 `AxB` が右詰めで描画されるように実装をお願いします．
連鎖アニメーションが終了したら，現在のスコアがまだ表示される状態に戻る感じです．

また，連鎖以外でスコアが加算される要素として，落下ボーナスというものがあります．
落下ボーナスはぷよを高速落下させることで発生するボーナスで，1マス毎（＋設置時）に1点入ります．
これも計算してスコア表示に反映させられるようにしてください．

## ゴースト表示のぷよが与えるフィールドに設置済みのぷよの表示に与える影響

操作中のぷよが現在の位置から落下させた場合にどこに落下するのかを表示したものがゴースト表示です．
ゴースト表示されたぷよが仮にそこに落下したとして，それによって連鎖が発生する連結数以上になる状態の場合，それらのフィールド上のぷよは若干明るい状態と若干くらい状態を軽く点滅するような形でハイライトされることで，次に置く手によってぷよが消えるのかどうかを把握することができるようになります．
例えば，フィールド上に赤ぷよが2つ連結した状態になっていて，組ぷよとして赤ぷよが2つあり，フィールド上で連結した赤ぷよ2つに対して隣接するようなところにゴースト表示がある場合，フィールド上で連結した赤ぷよ2つが軽く明滅する状態になるということです．
