# ぷよベース ゲームシステム詳細仕様書（現行実装準拠）

本書は，`main.py` / `src/` の**現在の実装挙動**を仕様として整理したものです．  
理想仕様ではなく，コード上で実際に動作している内容を優先します．

## 1. 概要

- 対象: 1P のぷよ操作・落下・連鎖・得点・描画・デバッグ表示
- 非対象（未実装）: 対戦システム，相殺，マージンタイム，おじゃま送受信
- ゲーム開始フロー:
1. 初期状態は `ready`（落下中組ぷよなし）
2. `START` 入力で `countdown`（3,2,1）
3. カウントダウン終了で `control` に遷移し，NEXT 先頭組を出現

## 2. アーキテクチャ

- `main.py`
  - Pygame ループ，入力取得，重力タイミング，補間進捗更新，描画呼び出しを担当．
  - CLI 引数 `-d/--debug` を解釈して描画モードを切替．
- `GameState`（`src/core/game.py`）
  - ゲーム進行の中心．状態遷移，移動/回転判定，接地ロック，連鎖処理，得点計算を担当．
- `Field`（`src/core/field.py`）
  - 盤面セル保持，落下処理，連結探索，消去を担当．
- `InputHandler`（`src/input_handler.py`）
  - エッジ入力（回転/開始/終了）とホールド入力（左右/下）を管理．
  - DAS とソフトドロップ連続入力生成を担当．
- `Renderer`（`src/ui/renderer.py`）
  - 盤面，組ぷよ，ゴースト，NEXT/NEXT2，スコア，演出，デバッグHUD描画を担当．

## 3. 座標系・盤面仕様

- 盤面サイズ: `GRID_WIDTH=6`, `GRID_HEIGHT=14`
- `y=0` が最下段，`y=13` が最上段．
- 可視段: `VISIBLE_HEIGHT=12`（`y=0..11`）
- オフスクリーン段: `y=12,13`（デバッグ表示時のみ盤面として可視化）
- 14段目仕様（現実装）:
  - 通常の組ぷよ配置判定 `can_place_pair` では，軸ぷよ `axis_y=13` は禁止．
  - 子ぷよは `sub_y=13` まで許可．
  - ただし回転専用判定で，補間中のみ一時的に `axis_y=13` を許容する特例あり（後述）．

## 4. ゲーム状態遷移

- `ready`
  - 初期状態．組ぷよなし．`START` で `countdown` へ．
- `countdown`
  - `COUNTDOWN_SECONDS=3.0` 秒．表示は `3 -> 2 -> 1`．
  - 終了時に `spawn_puyo()` 実行し `control` へ．
- `control`
  - プレイヤー操作中状態．移動/回転/重力/接地ロック処理を実行．
- `animate`
  - 連鎖演出状態．`resolve -> vanish_flash -> drop_tween -> resolve ...` を繰り返す．
  - 連鎖が途切れたら次組を `spawn_puyo()` して `control` へ戻る．
- `gameover`
  - 出現時窒息判定で遷移．操作不可．

## 5. 組ぷよ制御仕様

### 5.1 出現とNEXTキュー

- NEXTキューは常に2組以上維持．
- 出現時:
  - 軸: `current_puyo_1`
  - 子: `current_puyo_2`
  - 位置: `puyo_x=2`, `puyo_y=12`, `puyo_rot=UP`
- 表示上のNEXT/NEXT2は「上: `pair[1]` / 下: `pair[0]`」で描画され，出現向きと一致．

### 5.2 入力と優先制御

- キー割当:
  - `A`: 左移動
  - `D`: 右移動
  - `S`: ソフトドロップ
  - `←`: 左回転
  - `→`: 右回転
  - `Space` or `Enter`: 開始
  - `Q` or `Esc`: 終了
- DAS/連続入力:
  - 左右: 初回 `0.15s` 遅延後，`0.05s` 間隔
  - 下: `0.08s` 間隔
- 同時押し:
  - 左右同時ホールドは相殺（横移動なし）
  - 横移動と `DOWN` は同一フレームで適用可能
  - 左右同時ホールド中も `DOWN` は適用可能

### 5.3 90度回転

- まず通常回転（同一位置）を判定．
- 失敗時，軸下が詰まっていれば床キック（`y+1`）を試行．
- 床キック文脈では横キックにフォールバックしない．
- 軸下が詰まっていない文脈のみ，横キック（右→左）を試行．

### 5.4 180度回転

- 条件:
  - 現在縦向き（`UP` / `DOWN`）
  - 左右閉塞状態
  - 回転入力2回目（`右右/左左/右左/左右` いずれも可）
- 成立時は `UP <-> DOWN` の180度回転を試行．
- `UP -> DOWN` の180度回転では `y+1` の上昇回転を優先し，高い隣列を乗り越えられるようにする．

### 5.5 補間中の回転特例

- 回転専用判定は横移動用の「`y-1` スイープ判定」を使わない．
- `vertical_interpolation_progress > 0` かつ `axis_y == 13` のときのみ，
  一時的に上端配置を許可する特例がある．
- ただし `sub_y=14` になる向きは不許可（盤外）．

## 6. 接地ロック仕様

- 接地判定: `not can_move(0, -1, current_rot)`
- 接地中は毎フレーム:
  - `ground_frame_count += 1`
  - 非接地→接地の遷移時のみ `ground_contact_count += 1`
- ロック条件:
  - `ground_contact_count >= 8` または
  - `ground_frame_count >= 32`
- 補間中に床キック（上キック）した場合は，接地回数カウンタを追加で+1．

## 7. 重力と補間仕様

- 自由落下速度: `GRAVITY_INTERVAL_SECONDS=5`
- 自由落下補間:
  - `0.0 -> 0.5 -> 1.0` の 0.5 マス刻み
  - 描画オフセットは `vertical_interpolation_progress * PUYO_SIZE`
- ソフトドロップ補間:
  - `DOWN` ホールド中，しきい値経過で `0.5` マス進捗を反映
  - 自由落下補間と `max()` で合成
- 実マス移動は `GameState.update()`（手動）と `step_gravity()`（自動）で行う．
- UI ループの経過時間は単調時計を使って算出する．

## 8. 連鎖処理・演出仕様

- 連鎖演出状態:
1. `resolve`: まず重力落下を適用（必要なら `drop_tween` へ）
2. `vanish_flash`: 消去点滅 (`VANISH_FLASH_SECONDS=0.5`)
3. `drop_tween`: 落下補間 (`CHAIN_DROP_TWEEN_SECONDS=0.24`)
4. 再び `resolve` へ
- 点滅周期: `VANISH_BLINK_INTERVAL_SECONDS=0.08`
- 消去候補がなくなった時点で連鎖終了，次組出現へ遷移．
- `resolve` は時間を持たない内部遷移として扱い，演出終了直後に不要なフレーム待ちを挟まない．

## 9. 得点仕様

- 1連鎖ごとに次式:
  - `消去個数 * max(1, 連鎖ボーナス + 連結ボーナス + 色数ボーナス) * 10`
- 連鎖ボーナス:
  - `CHAIN_BONUS_TABLE` を使用（19連鎖以上は512）
- 連結ボーナス:
  - `get_connection_bonus()` を使用（4:0, 5:2, ..., 10:7, 11以上:10）
- 色数ボーナス:
  - `COLOR_BONUS_TABLE` を使用（1:0, 2:3, 3:6, 4:12, 5:24）
- おじゃま送信点・マージンタイム関連は未実装．

## 10. 消去・落下の制約

- 連結探索/消去対象は可視12段（`y=0..11`）のみ．
- `y=12,13` は連結判定および消去対象外．
- `Field.drop_puyo()` は `y=0..12` のみ再構成し，`y=13` は落下させない．
- おじゃま巻き込み消去も可視12段内隣接のみ．

## 11. 描画仕様

- 共通描画:
  - フィールド枠，グリッド，スコア，NEXT/NEXT2，組ぷよ，ゴースト表示
- ゴースト:
  - セル中心基準の `65%` サイズ矩形，半透明表示
- 通常モード（`python3 main.py`）:
  - 可視12段のみ描画
- デバッグモード（`python3 main.py --debug`）:
  - 13/14段を含む14段表示
  - `OFFSCREEN (13-14)` ラベル表示
  - フィールド下に `Ground F` / `Ground C` HUD を表示

## 12. 既知の制約・未実装項目

- 対戦機能は未実装（プレイヤーは単一フィールドのみ）．
- おじゃま送受信，相殺，マージンタイム未実装．
- 乱数シード固定やリプレイ機能は未実装．
- UI/演出は機能実装優先であり，実機完全再現ではない．
